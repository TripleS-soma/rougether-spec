# 프로덕트 개요 (PRD)

출처: AI·SW마에스트로 제17기 프로젝트 기획서 (원제 **루틴빌리지** → 현재 명칭 **루게더**). 팀 tripleS (장진형·임채영·최준서).

## 한 줄 정의

**집과 커뮤니티로 성장하는 AI 루틴 관리 앱.** 일상 루틴 수행을 캐릭터·개인 방·공동 집의 성장으로 시각화하고, 지인 기반 공동체와 AI 코칭으로 지속 가능한 습관 형성을 돕는다.

기술 키워드: Cross-Platform Mobile App · LLM · Computer Vision · Gamification.

## 문제 정의

- 운동·독서·공부 등 자기관리를 시도하지만 바쁜 일정·피로·낮은 강제성·즉각 보상 부족으로 루틴을 장기 유지하지 못한다.
  - 행동 자동화에 18~254일(중앙값 66일, Lally 2010). 새해 결심 6개월 지속 성공률 46%(Norcross 2002).
- 하루 단위로는 성취감이 약하다 — "오늘도 성실히 살았다"가 눈에 보이는 결과로 남지 않는다.
- 고립된 환경의 습관은 사회적 지지를 받는 습관보다 쉽게 무너진다. 지인 기반 공동체 + 사회적 책임감을 활용한 사례는 시장에 드물다.
- 기존 앱은 체크리스트·통계 중심이라 성취감이 약하다.

## 기획 의도 (3가지 핵심 방향)

1. **루틴 완료 보상 시각화** — 루틴 완료 보상으로 개인 방 꾸미기·캐릭터 성장. 장기 효과를 매일 시각적으로 확인.
2. **공동체 책임감 활용** — 지인 기반 '집' 구성, 단체 미션 수행, 공동 보상. 고립된 활동을 함께 성장하는 경험으로 확장.
3. **AI 기반 맞춤 코칭** — 루틴 패턴 분석으로 맞춤 추천·리마인드·주간 회고·인증사진 보조 판단.

## 타겟 유저

대학생·취준생 등 소규모 공동체. 자격증 스터디(공부 인증), 운동 동아리, 취업 준비 그룹 등 **같은 목표를 공유하는 지인 단위**. 향후 학교·회사·취미 그룹으로 확장.

## 차별점 (시장 분석)

| 서비스 | 강점 | 한계 |
| --- | --- | --- |
| 루빗 | 캐릭터·방 꾸미기, 통계 | 개인 성장 중심, 공동체 집 성장 한계 |
| 마이루틴 | 실용적 루틴 관리, 스트릭 | 시각 보상·커뮤니티 약함 |
| 투두메이트 | 친구 반응, 크루 | 공간 성장·보상 경제로 연결 안 됨 |
| Forest | 집중 시간 측정, 나무 성장 | 생활 루틴·공동체 성장 한계 |

→ **루게더의 차별**: 지인 기반 소규모 공동체의 **'집' 단위 공동 성장**과 **AI 기반 루틴 실패 원인 분석**을 하나의 흐름으로 연결.

## 핵심 경험 루프

```
루틴 완료(체크형/사진 인증형)
  → 코인 보상 + 스트릭 갱신
  → 개인 방·캐릭터 성장 시각화
  → 집 단체 미션 기여 → 집 경험치/레벨·외관 성장
  → AI가 패턴 분석 → 맞춤 추천·주간 회고
```

## MVP 범위 (우선순위)

도메인별 기능은 [features.md](features.md), 데이터는 [erd.md](erd.md) 참조.

> 멘토 피드백("기능 나열보다 과감한 덜어내기")을 반영해 우선순위로 쪼갠 **초안**이다. 단계별 배치는 팀 확정이 필요하다.

**P0 — 핵심 루프** (없으면 제품이 성립 안 함)
- **소셜 로그인 / 회원가입**(카카오·구글·애플) — 인증/인가
- 온보딩(목표·캐릭터 선택, 최소)
- 루틴/투두·카테고리 관리 → 오늘 현황 → 완료 체크 → 코인 보상 + 스트릭
- 개인 방 성장·아이템 배치(기본)
- 집 참여(초대코드)·단체 미션(기본)

**P1 — 보상 / 공동체 심화**
- 상점·인벤토리·다이아 구매, 뽑기
- 사진 인증형 루틴(업로드·공개 범위)
- 집 탐색/관리/구성원/방명록, 집 레벨·테마 보상
- 방 스냅샷 공유

**P2 — AI 고도화** (순차)
- 맞춤 루틴 추천 · 주간 회고 생성 · 리마인드 · 인증사진 보조 판단 · 단체 미션 추천

**초기 비범위**: private upload, 결제/구독 정산.

## 성공 지표 (KPI)

DAU · 루틴 완료율 · 7일 재방문율 · 집 참여율 · 단체 미션 참여율.

## 수익 모델

광고 · 구독제 패스(고급 테마, AI 리포트) · 재화 구매(꾸미기 아이템, 랜덤박스). 향후 학교·커뮤니티 단위 습관 챌린지 플랫폼으로 확장.

## 기술 개요

React Native 앱 → Spring Boot API 서버 → MySQL(RDBMS) + S3 Object Storage + FCM Push + OpenAI API. Scheduler/Batch Worker가 리마인드 타이머·주간 회고 생성·스트릭 정리를 담당. (아키텍처 상세는 서버 repo `docs/architecture/`)

- AI 모델 구분: 추천·주간 회고·단체 미션 추천 = **소형 LLM**, 짧은 리마인드 문구 = **경량 LLM**, 인증사진 보조 판단 = **비전(멀티모달) 모델**, 예외/고품질 데모 = **상위 LLM**. (구체 모델·가격·정책은 변동 가능하므로 PRD에 고정하지 않는다 — 모델 선정 근거는 기획서/기술 문서에서 관리)

## 멘토 피드백 → 방향성

- **박영용**: MVP는 기능 나열보다 과감한 **'덜어내기'**. 부가 꾸미기보다 유저를 Lock-in 시킬 **단 하나의 킬러 피처**와 그 이면의 데이터 아키텍처. 루틴 **실패 패턴 분석 기반 초개인화**가 핵심 가치.
- **유용태**: 루틴을 **시작·지속하기 어려운 사용자에게 실질적 도움**이 되는지가 관건. 사용 흐름 + 누적 보상 체계의 세밀한 설계.
- **강한규**: 루틴 = 소속감·재미로도 표현 가능. 가족·취미·학교·회사 등 **그룹 확장성**. 알라미(딜라이트룸)처럼 "루틴이 가치가 되는 지점" 고민.

→ 여기서 파생되는 결정(킬러 피처 정의 등)은 [open-questions.md](open-questions.md).

## 확정된 핵심 설계 (기획서 근거)

- **다중 집 가입 허용**: "사용자는 하나 이상의 집에 참여할 수 있다." → `house_members`는 같은 집 중복 가입만 막고(unique `(house_id, user_id)`), 한 사용자가 여러 집에 속할 수 있다. ([erd.md](erd.md) 반영)
- **재화 2종 분리**: 코인(루틴 실천 보상) / 다이아(아이템 구매). 보상 경제는 초기엔 단순 규칙으로.
