# 루틴 / 투두 — PRD

상위 문서: [product.md](../../product.md) · [features.md](../../features.md) · [erd.md](../../erd.md) · [api.md](../../api.md)

## 한 줄 정의

사용자가 매일 실천할 **루틴**(반복)과 **투두**(단발)를 카테고리로 묶어 관리하고, 완료를 기록하면 **코인 보상 + 스트릭 갱신**으로 이어지는 도메인. 루게더 핵심 경험 루프의 **시작점**이다.

## 왜 필요한가 (문제)

- 운동·독서·공부 등 자기관리를 시도하지만 바쁜 일정·피로·낮은 강제성·즉각 보상 부족으로 **루틴을 장기 유지하지 못한다**. (행동 자동화 중앙값 66일, 새해 결심 6개월 지속 46%)
- 하루 단위로는 성취감이 약하다 — "오늘도 성실히 살았다"가 눈에 보이는 결과로 남지 않는다.
- 기존 앱은 체크리스트·통계 중심이라 성취감이 약하다.

## 이 도메인이 제공하는 가치

- **완료의 즉각 보상화**: 루틴/투두 완료 시 코인을 지급하고 스트릭을 갱신해, "오늘의 실천"을 즉시 손에 잡히는 보상으로 바꾼다. 이 코인·스트릭이 방·캐릭터·집 성장(타 도메인)의 입력이 된다.
- **두 가지 인증 방식**: 가벼운 **체크형**과, 사회적 책임감·기록을 강화하는 **사진 인증형**을 함께 제공한다. 사진 인증은 공개 범위(카테고리 `visibility`와 같은 값 집합: 비공개/친한친구/집/공개)를 사용자가 통제한다.
- **연속성의 시각화**: 스트릭(연속 성공일)으로 "끊기지 않았다"는 동기를 만든다.
- **AI 코칭의 데이터 기반**: 루틴 수행/실패 기록(`routine_logs`)과 인증 사진(`photo_verifications`)이 패턴 분석·맞춤 추천·주간 회고의 원천 데이터가 된다.

## 핵심 시나리오

1. **설정**: 사용자가 카테고리(예: 운동)를 만들고, 그 안에 루틴("아침 스트레칭", 매주 월·수·금 07:00, 체크형)과 투두("러닝화 사기", 마감일 지정)를 등록한다.
2. **오늘 현황**: 앱을 열면 오늘 해야 할 루틴·투두가 카테고리별로 보이고, 완료 수·남은 수·전체 진행률이 한눈에 보인다. 루틴은 시간순 정렬.
3. **완료**: 루틴을 완료 체크하면 수행 시간이 기록되고 코인이 지급되며 스트릭이 갱신된다. 사진 인증형 루틴은 카메라/앨범에서 사진을 올려 인증한다.
4. **취소**: 잘못 체크한 경우 **당일 내** 완료를 취소하면 지급 코인·스트릭이 롤백된다.
5. **사진 관리**: 인증 사진의 공개 범위를 바꾸거나, 사진만 삭제(루틴 완료 상태는 유지)할 수 있다.

## 경험 루프에서의 위치

```
[이 도메인] 루틴/투두 완료(체크형/사진 인증형)
  → 코인 보상 + 스트릭 갱신          ← 이 도메인이 생성
  → 개인 방·캐릭터 성장 시각화        → 방 도메인
  → 집 단체 미션 기여 → 집 성장        → 집 도메인
  → AI 패턴 분석 → 맞춤 추천·주간 회고  → AI(순차 고도화), 입력은 routine_logs/photo_verifications
```

## 담당 범위

- **table**: `categories`, `routines`, `routine_logs`, `photo_verifications`, `todos`, `streaks`
- **기능**: 카테고리 관리, 루틴 관리(인증 방식·반복 조건), 투두 관리, 오늘 현황, 루틴/투두 완료 체크·취소, 사진 인증(등록·공개범위·삭제). 상세는 [features.md](features.md).

## 의존성 (타 도메인)

- **재화 지급/차감**: 완료 보상 코인은 `user_wallets`(회원/재화 도메인)에 반영된다. 이 도메인은 `routine_logs.reward_currency_type` / `reward_amount`, `todos.reward_*`에 보상 사실을 기록하고, 지갑 반영은 쓰기 트랜잭션으로 함께 묶는다.
- **사진 공개 범위 "집 구성원 공개"**: 노출 대상은 집(공동) 도메인(`house`, `house_members`)에 의존한다. 이 도메인은 `photo_verifications.privacy_scope`만 저장한다.
- **AI 분석**: `photo_verifications.ai_review_status`, `routine_logs` 패턴 분석은 AI(순차 고도화) 영역. 이 도메인은 입력 데이터와 동의 상태만 보관한다.
- **방/집 성장 반영**: 코인·스트릭을 입력으로 쓰는 성장 로직은 방·집 도메인 소관.

## 미결정

- 완료 취소 "당일" 기준 타임존(서버 UTC vs KST), `repeat_type`/`repeat_days` 값 스펙, 코인 보상 금액 규칙, 공개 범위 단위(루틴별 vs 카테고리별)는 [open-questions.md](../../open-questions.md) 참고.
