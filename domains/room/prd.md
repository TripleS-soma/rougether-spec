# 개인 방 PRD

루게더의 핵심 방향1 — **루틴 완료 보상을 개인 방의 성장으로 시각화**한다. "오늘도 성실히 살았다"가 눈에 보이는 결과물(레벨업·꾸며진 방)로 남게 만드는 도메인이다.

## 목적

체크리스트·통계 중심 앱은 하루 단위 성취감이 약하다. 개인 방은 루틴 달성을 **방 레벨·성장 포인트**와 **꾸미기 결과물**로 환산해, 누적된 성실함을 매일 시각적으로 확인시키는 보상 표면이다.

## 사용자 가치

- **누적의 가시화**: 매일의 루틴 완료가 방 `growth_level` 상승으로 쌓인다. 통계 숫자가 아니라 자라나는 공간으로 보인다.
- **자기표현**: 보유 아이템을 슬롯에 배치해 자기만의 방을 만든다. 꾸미기 동기가 루틴 지속 동기로 연결된다.
- **연속성의 압박과 보람**: 스트릭(연속 성공일)을 방에서 확인해 "끊기지 않게 하고 싶다"는 행동 강화를 만든다.
- **공유**: 완성된 방을 스냅샷으로 남겨 외부 공유 또는 집 구성원에게 보여준다. 성장의 사회적 인정.

## 핵심 시나리오

1. **성장 확인**: 사용자가 오늘 루틴을 완료하면 보상이 방 성장 포인트로 누적되고, 임계치를 넘으면 `growth_level`이 오르며 시각 피드백이 뜬다.
2. **방 꾸미기**: 인벤토리(보유 아이템)에서 아이템을 골라 방의 슬롯(`slot_type`)에 배치하고 저장한다. 저장된 배치가 다음 방문 시 그대로 보인다.
3. **스트릭 표시**: 방 화면에서 현재 연속 성공일과 최장 기록을 확인한다.
4. **스냅샷 공유**: 현재 방 상태를 이미지로 저장해 외부 공유하거나 집 구성원에게 공개한다.

## 범위

이 도메인은 `personal_rooms`(1:1 user 소유, `growth_level`)와 `room_surface_slots`(슬롯별 배치)를 소유한다. 스트릭은 표시 목적으로 `streaks`를 **읽기만** 한다.

## 의존성 (다른 도메인)

- **루틴/투두**: 방 성장 포인트와 스트릭의 원천. 루틴 완료 보상이 어떤 규칙으로 `growth_level`/성장 포인트에 반영되는지는 루틴 도메인과 합의 필요 — **미정**.
- **상점/인벤토리**: 배치 가능한 아이템(`user_items`, `items`)의 보유·슬롯 타입 정의. 슬롯에 넣을 수 있는 아이템 검증은 인벤토리 계약에 의존.
- **집(공동)**: 방 스냅샷의 "집 구성원 공개" 대상. 방명록(`room_guestbooks`)은 집 도메인 담당 — 이 문서에서 다루지 않는다.

## 미결정 사항

- 방 slot 목록과 필수 slot 기준(`slot_type` enum 값 집합).
- `wallpaper`/`floor`를 slot으로 볼지 room theme field로 볼지.
- 루틴 보상 → `growth_level`/성장 포인트 환산 규칙. `personal_rooms`에 성장 포인트 누적 컬럼이 별도로 필요한지(현재 ERD는 `growth_level`만 보유).
- 스냅샷 이미지의 저장 위치·`*_key` 발급 주체(별도 table 없음).
